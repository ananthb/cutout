Setup Guide

Deploy your own Cutout instance in a few minutes.

Prerequisites

  • A Cloudflare account with a domain
  • Nix installed (for the dev environment)
  • Email Routing enabled on your domain
  • Cloudflare Access configured (protects the management UI)

1. Clone and enter the dev shell

git clone https://github.com/ananthb/cutout.git
cd cutout
nix develop

2. Create Cloudflare resources

# Create KV namespace
wrangler kv namespace create KV

Copy the namespace ID into your wrangler config.

3. Configure Email Routing

In your Cloudflare dashboard, set up a catch-all rule that sends all email to the Cutout worker.

4. Configure Cloudflare Access

Create an Access application for the /manage path to protect the management UI. Set CF_ACCESS_TEAM and CF_ACCESS_AUD in your wrangler config.

5. Deploy

wrangler deploy

6. Manage aliases

Visit /manage to create and manage aliases through the HTMX-based admin UI.
